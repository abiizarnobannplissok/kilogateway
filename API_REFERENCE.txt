PENJELASAN: BASE URL FORMAT
----------------------------

Di OpenAI-compatible API, ada 2 cara penulisan Base URL:

1. BASE URL SAJA (tanpa endpoint)
   Contoh: https://kilogateway.vercel.app/v1

   Ini cara yang PALING UMUM dipakai.
   SDK (OpenAI Python, OpenAI Node.js, dll) akan otomatis
   menambahkan endpoint-nya sendiri:
   - /v1/chat/completions
   - /v1/models
   - /v1/embeddings

   Jadi kalau pakai SDK, cukup tulis Base URL-nya saja.


2. BASE URL LENGKAP (dengan endpoint)
   Contoh: https://kilogateway.vercel.app/v1/chat/completions

   Ini dipakai kalau:
   - Pakai curl langsung (tanpa SDK)
   - Platform tertentu yang minta URL lengkap
   - Testing manual

   Tapi kalau pakai SDK, JANGAN pakai cara ini.
   SDK akan gagal karena URL-nya sudah ada endpoint-nya.


KESIMPULAN:
-----------
- Pakai SDK?       -> Gunakan: https://kilogateway.vercel.app/v1
- Pakai curl?      -> Gunakan: https://kilogateway.vercel.app/v1/chat/completions
- Platform minta?  -> Lihat dokumentasi platform tersebut

Kebanyakan platform (OpenAI, Anthropic, dll) menggunakan format "/v1"
sebagai Base URL, dan SDK akan menangani sisanya.


ENDPOINT YANG TERSEDIA
-----------------------
GET  /health                    -> Cek status server
GET  /v1/models                 -> Daftar semua model
POST /v1/chat/completions       -> Kirim pesan ke AI


STATUS CODE ERROR
-----------------
401 -> API Key salah atau tidak ada
400 -> Format request salah
404 -> Model tidak ditemukan
500 -> Error di server
504 -> Timeout (terlalu lama)


TIPS
----
1. Gunakan model FREE untuk testing dan development
2. Gunakan model BERBAYAR untuk production
3. Set max_tokens sesuai kebutuhan (hemat token = hemat biaya)
4. Streaming (stream: true) untuk response yang panjang
5. Temperature 0.7 untuk hasil yang seimbang
6. Temperature 0 untuk hasil yang konsisten
7. Temperature 1 untuk hasil yang kreatif
